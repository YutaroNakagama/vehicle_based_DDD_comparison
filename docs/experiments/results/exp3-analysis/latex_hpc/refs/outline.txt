3. DDD Based on the Vehicle Dynamics Data
    - 概要
        - DDDの定義
            - 入出力 (2値分類問題)：車両情報 -> Drowsy／Awake    
            - 車両モデル
            - パラメタ: 時間窓，周波数，他
        - DDDのフローチャート
    - A. ML pipeline Configuration
        - DDDのパイプライン
    - B. MMDAP Dataset
        - データの概要（Aygun）
        - 実験設定
        - DDDにおける正解ラベル: EEG，DRT
    - C. DDD methods
        - 手法の概要
            - (再現手法) (表3で比較)
                1. SVM_wavelet (Zhao)
                2. SVM_ANFIS (Afernezhad) (4 Index, ANFIS)
                3. LSTM_DRT (Wang)
            - (統合手法)
                - RF 
        - (1) SvmA
            - 特徴量：ステアリング角度／角速度から時間方向統計量 
            - 特徴量選択：ANFISによるWrapper法
            - 分類器：SVM
            - 正解ラベル：元論文はKSS，本研究ではEEG
        - (2) SvmW
            - 特徴量：ステアリング角度からWavelet分解 
            - 特徴量選択：なし
            - 分類器：SVM
            - 正解ラベル：元論文は実験毎の主観評価，本研究ではEEG
        - (3) Lstm
            - 特徴量：5つの車両情報から3つの特徴量 
            - 特徴量選択：T検定
            - 分類器：Attention付双方向LSTM
            - 正解ラベル：元論文は電話応答タスク，本研究ではDRT 
        - (4) RF
            - 特徴量：5つの車両情報から過去DDD全ての特徴量 
            - 特徴量選択：一変量統計（Univariate Method）による特徴量選択
            - 分類器：Random Forest
            - 正解ラベル：EEG
    - D. 実装
        - 共通環境
            - Python 3.10.12
            - scikit-learn 1.5.2（SVM, RF）
            - TensorFlow 2.18.0 / Keras 3.6.0（LSTM）
            - pyswarm 0.6（PSOの実装）
            - Optuna 4.1.0（ハイパーパラメータ調整）
        - モデル別の実装
            - SVM, RF → scikit-learn
            - LSTM → TensorFlow / Keras
            - SvmAのPSO最適化 → pyswarm
        - Optunaを用いたハイパーパラメータ調整

4. Experiment to Check the Reproducibility
    - 研究課題
        - RQ1: 過去に報告された性能メトリックが再現できるか？
        - 実験手法:
            - 既存手法を再実装し、RVsに近い結果が得られるか評価
            - C1: 元論文に近い設定
            - C2: すべての手法で共通の設定
        - SvmAの再現性検証
            - 設定（C1）
                - 評価指標: AUC、Accuracy、# Features（特徴選択数）
                - データ分割: 元論文に記載なし →
                - 訓練精度が報告されていると仮定し、データ分割なし
                - ラベル: 元論文ではKSS、本研究ではEEGに基づく
                - 使用データ: 元論文は20時間36分、本研究では164時間分を使用
            - 考察
                - C1の結果はRVsと近かった
                - C1とC2の間には大きな差異があり、C1は訓練データでの精度のみを最適化していた可能性あり（過学習）
                - C2ではテスト精度が低下し、一般化性能の低さが判明
        - SvmWの再現性検証
            - 設定（C1）
                - 評価指標: "Correct Rate"（正確率）をAccuracyと解釈
                - データ分割: 訓練:テスト = 3:7
                - ラベル: 元論文では自己評価、本研究ではEEGラベルを使用
                - データ処理: スライディングウィンドウなし、約800秒のデータを一括処理
                - サンプリングレート: 60 Hz
            - 考察
                - C1とC2の両方でRVsより低い精度を記録
                - 原因: 元論文で使用されたラベル（自己評価）と本研究のEEGラベルの違い
                - C2の結果の方がさらに低く、標準的なデータ分割（8:1:1）と精度指標（テストセット評価）が影響
        - C. Lstmの再現性検証
            - 設定（C1）
                - 評価指標: Precision、Recall、F1-score（元論文準拠）
                - データ分割: 5-fold クロスバリデーション
                - ラベル: DRTイベントの有無を基に分類
                - ウィンドウサイズ: 10秒（オーバーラップなし）
                - サンプリングレート: 10 Hz
            - 考察
                - C1の結果はRVsより大幅に低かった
                - 原因の一つ: 元論文とMMDAPデータセットの違い（Fig. 4 で信号の傾向が異なることを確認）
                - C1では100%のRecallを達成するが、Precisionが低下し、False Positiveが増加
                - C2ではEEGを基にしたラベルを使用 → モデルの性能が大きく変動
        - D. まとめ
            - RVsの完全な再現は不可能だった
            - 原因:
                - 元論文の評価指標が訓練データの性能を報告していた可能性
                - ラベリング方法の違い（自己評価 vs EEG）
                - データセットの違い（MMDAPの特徴 vs 元論文のデータ）
            - 示唆:
                - 標準化された評価手法の確立が必要
                - 訓練・テストの区別を明確にするべき

5. Experiment for Performance Comparison
    - 研究課題  
        - RQ2: もっとも性能が高い手法は？
        - 方法:
            - C2設定（統一した評価基準）で4つの手法（SvmA, SvmW, Lstm, RF）を比較
            - 第一実験で評価した3手法に加え、RFを評価
            - 標準的な性能指標（Accuracy, Precision, Recall, F1-score）で比較
    - A. 結果
        - Tab. V: 4つの指標（Accuracy, Precision, Recall, F1-score）を各手法で比較
        - Fig. 5: 各手法のROC曲線を図示
        - 主な結果:
            - RFが全体的に最も優れた性能を示した
            - LstmはPrecisionが最も高いが、Recallが低いため実用性が低い可能性
    - B. 考察
        - RFの優位性
            - RFは他の手法より多くの特徴量を活用できたため、高い精度を達成
        - SvmAの過学習
            - ANFISを用いたSvmAは訓練精度が高いものの、テスト精度が低下 → 過学習の可能性
            - RFはこの問題を回避し、より汎化性能を確保
        - SvmWのデータ処理方法の影響
            - SvmWはデータ全体を一括処理 →他の手法のように時間ウィンドウを考慮していない
            - 短期間の変動を捉えにくくなり、精度が低下
        - Lstmの課題
            - Wang et al.のデータでは良好な精度を示したが、本研究のデータセットでは信号変化が少なく、適切にパターンを学習できなかった可能性
            - Lstmにはより長い時間ウィンドウが必要だった可能性
        - C. Threats to Validity（妥当性に関する脅威）
            - ハイパーパラメータ最適化の影響
                - RFはデフォルト設定でも高性能を発揮
                - LSTMやSVMは適切なハイパーパラメータ調整が必要 → 最適化不足の可能性
            - RFの特徴量バイアス
                - RFはすべての特徴量を利用できるため、他の手法と不公平な比較になっている可能性
                - 他の手法も同じ特徴量を使用すれば、性能が向上する可能性
            - データセットの一般化可能性
                - 本研究のデータセットはシミュレーターでの直線道路運転 → 現実の運転条件とは異なる
                - カーブや加減速の頻繁な運転シナリオでは、モデルの性能が異なる可能性
            - 個人差の影響
                - 被験者ごとの運転特性や眠気の現れ方が異なる
                - 個人ごとの違いを考慮すれば、モデル精度が向上する可能性あり
            
6. Conclusion
    - 主な貢献
        - DDD分野における「再現性」と「標準化された開発プロセス」の重要性
        - 共通フレームワークを用いた再現実験を実施し、元論文の結果と大きく異なるケース
        - 再現性の問題を明確化、評価手法の標準化の必要性
    - 主要な成果
        - RFベースの新しい軽量DDD手法を開発
        - 既存の全特徴セットを統合したアプローチ
        - 標準化された評価基準を用いた比較実験で最も高い性能を発揮
    - 今後の課題
        - 大規模なMLモデルを活用したDDD手法の開発
        - 多様な運転環境・データセットを用いた追加評価実験
        - 実車両への適用と運用 → 疲労関連事故の削減
